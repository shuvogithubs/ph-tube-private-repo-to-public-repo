Discuss the scope of var, let, and const

var .

  • Function Scope:Variables that are declared inside a function are called local variables and in the function scope. Local variables are accessible anywhere inside the function.
  • Block scope: Variable that is declared inside a specific block & can't be accessed outside of that block.

let .

  • Block Scope: Let is used to declare a variable that is block scoped in JavaScript. In JavaScript, the var keyword is typically used to declare a variable that is treated like any other variable, whereas variables declared using the let in JavaScript are block scoped.
  • Hoisting: Let variables are raised to the top of their contained block like var variables are, but they are not initialized. A ReferenceError will be raised if a let variable is attempted to be accessed before it has been declared.

const .

  • Block Scope:the const keyword in JavaScript to declare variables whose value can be initialized only at the time of declaration. Like let, variables declared with const are also block-scoped.
  • Immutability: After being given a value, const variables cannot be changed. If the value they hold is an object or an array, it may nevertheless be modifiable. This means you can change a variable's assignment but not its properties, or the elements of an array declared with const.
Tell us the use cases of null and undefined

undefined .

  • Implicit Default Value: Undefined is automatically assigned to a variable when it is declared but not given a value.
  • Function Parameters: The value of a parameter inside a function will be undefined if it is expected but not supplied when the function is called. Using this, you can make some function parameters optional.
  • Array Elements: If you access an array element at an index that is out of bounds or if an element has not been assigned a value, you will get undefined.
  • Object Properties: Undefined is returned if you attempt to access an object attribute that doesn't exist.

null .

  • Explicit Absence of Value: Null is frequently used to signify clearly that no object value exists or that a variable or object property should be empty. It's a way to show that a variable has been purposefully set to zero.
  • Cleaning Up References: In some circumstances, setting an object or variable to null will allow you to free up memory or other resources. Particularly in applications with manual memory management, such as some JavaScript frameworks or libraries, this can be crucial.
  • Comparison: When determining whether something has been purposefully set to nothing, you can use null as a deliberate value to compare to or check against.
What do you mean by REST API?

REST API .

Representational State Transfer Application Programming Interface, also known as REST API, is a set of architectural guidelines and restrictions for creating networked applications. It is a method for developing web services that enables communication between various software programs via the internet or within an intranet.


Key principles and characteristics of RESTful APIs include:

  1. Stateless,
  2. Client-Server Architecture,
  3. Resource-Based,
  4. Uniform Interface,
  5. Statelessness,
  6. Representation,
  7. Stateless Communication,
  8. HTTP Methods,
  9. Status Codes,
  10. HATEOAS (Hypermedia as the Engine of Application State).
